# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-11-15

### Added
- Initial release of Wich compliance checker
- 100-point scoring system across 10 categories
- Legal & Licensing checks (LICENSE, OSI approval, copyright)
- Documentation checks (README, SECURITY.md, CONTRIBUTING.md)
- Code Quality checks (linting, formatting, .gitignore, pre-commit)
- Security checks (dependency scanning, secrets protection, auth docs)
- Governance checks (Code of Conduct, templates, maintainers)
- CI/CD checks (automated builds, tests, security scanning, deployments)
- Testing checks (unit tests, integration tests, coverage)
- Performance checks (benchmarks, optimization, scalability)
- Logging & Monitoring checks (logging frameworks, observability)
- Community Health checks (responsible disclosure, support, changelog)
- Command-line interface with verbose and JSON output modes
- Comprehensive README with usage examples
- MIT License
- Security policy and responsible disclosure process
- Contributing guidelines
- Code of Conduct
- Grading system (A-F) based on score percentage

### Features
- Language-agnostic repository analysis
- Case-insensitive file detection
- Detailed category breakdowns
- Actionable recommendations
- CI/CD integration examples
- Zero external dependencies (Python 3.7+ only)
